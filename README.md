# 🧪 End-to-End Manual Testing Project – Food Ordering Application

## 📌 Project Overview
This project demonstrates **end-to-end manual testing** of a food ordering (pizza delivery) application.

It includes:
- Detailed test scenarios
- Step-by-step test cases
- Real-time bug reporting with severity & priority

The goal is to validate the application’s functionality, usability, and overall user experience.

---

## 🎯 Testing Scope

The application flow tested includes:

- User Login (Mobile Number & OTP)
- Home Page Navigation
- Search & Menu Browsing
- Add to Cart Functionality
- Apply Coupon Codes
- Address Selection
- Checkout Process
- Order Placement & Confirmation

---

## 🛠️ Tools Used

- Google Sheets (Test Case & Bug Tracking)
- Manual Testing Techniques
- GitHub (Documentation)

---

## 🧪 Test Case Coverage

The project covers:

### ✅ Functional Testing
- OTP verification (valid/invalid cases)
- Button functionality (Submit, Cancel, Skip, Continue)
- Search and filter features
- Add to cart & remove items
- Checkout flow

### ✅ UI Testing
- Button visibility and alignment
- Navigation flow
- Page redirection

### ✅ Negative Testing
- Invalid mobile numbers
- Incorrect OTP
- Missing inputs
- Invalid coupon codes

### ✅ End-to-End Testing
- Complete order journey from login to delivery confirmation

---

## 📋 Test Case Structure

Each test case includes:

- Test Case ID  
- Description  
- Pre-conditions  
- Test Steps  
- Expected Result  
- Actual Result  
- Status (Pass/Fail)

## 🐞 Bug Report Summary

Bugs were identified and categorized based on:

- **Bug ID**
- **Description**
- **Severity (High/Medium/Low)**
- **Priority**
- **Type (Functional / UI / Usability)**
- **Steps to Reproduce**
- **Screenshots (Google Drive links)**

## 🔍 Key Bugs Identified

- OTP validation issues  
- Application crashes during login  
- UI misalignment in some sections  
- Language inconsistency issues  
- Missing validations in forms  
- Navigation & redirection issues

## 📊 Sample Bug Types
